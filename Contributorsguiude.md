# Contributing to AfroPay Stellar

Thank you for considering contributing to AfroPay Stellar.

AfroPay Stellar is a cross-border remittance platform built on the Stellar blockchain focused on fast, affordable, and secure global payments across Africa and emerging markets.

We welcome developers, designers, technical writers, blockchain contributors, and open-source enthusiasts of all skill levels.

---

# Project Vision

Our goal is to build decentralized financial infrastructure that makes cross-border payments:
- Faster
- More affordable
- More accessible

We believe blockchain technology and the Stellar ecosystem can help improve financial inclusion globally.

---

# Getting Started

## 1. Fork the Repository

Click the Fork button at the top right of the repository page.

Then clone your fork locally:

```bash
git clone https://github.com/YOUR_USERNAME/afropay-stellar.git
cd afropay-stellar
````

---

## 2. Install Dependencies

```bash
npm install
```

or

```bash
yarn install
```

---

## 3. Run the Development Server

```bash
npm run dev
```

The app should now run locally at:

```bash
http://localhost:3000
```

# Beginner-Friendly Contributions

Look for issues labeled:

```bash
good first issue
beginner friendly
help wanted
```

These are suitable for new contributors.

---

# Branch Naming Convention

Please use descriptive branch names:

```bash
feature/add-wallet-modal
fix/navbar-bug
docs/update-readme
```

---

# Code Standards

Please follow these guidelines:

* Use TypeScript
* Write clean and readable code
* Keep components reusable
* Follow the existing project structure
* Use meaningful variable names
* Avoid unnecessary dependencies

---

# UI/UX Guidelines

AfroPay Stellar follows a modern Web3 design system:

* Dark mode first
* Glassmorphism
* Responsive layouts
* Smooth animations
* Clean fintech aesthetics

Maintain consistency with the existing UI.

---

# Commit Message Convention

Examples:

```bash
feat: add wallet connection modal
fix: resolve transaction history bug
docs: update installation guide
style: improve dashboard responsiveness
```

# Pull Request Process

Before submitting a PR:

* Ensure your code builds successfully
* Test your changes locally
* Keep pull requests focused
* Add clear descriptions

PR checklist:

* [ ] Code tested locally
* [ ] No build errors
* [ ] Follows project structure
* [ ] Documentation updated if needed
